## 2.0.0 - 2026-03-30
* The SDK now supports configurable logging through ClientOptions.Builder.logging(), allowing developers to debug HTTP requests and responses (silent by default). Added new SSE event discrimination capabilities for parsing Server-Sent Events with event-level unions. Removed deprecated INSULIN_INJECTION enum value and InputStream-based lab report upload methods.
* The `parserCreateJob(InputStream, String)` method overloads have been removed from `RawLabReportClient`. Replace calls with the existing `parserCreateJob(File, BodyCreateLabReportParserJob)` method which accepts a File instead of InputStream. New optional parameters are now available for lab test configuration: `lab_account_id` and `lab_slug` in `CreateLabTestRequest`, `lab_slug` in `LabTestsGetMarkersRequest`, and `appointment_notes` in `RequestAppointmentRequest`.
* The SDK removes the `teamId` parameter from bulk operations methods and adds new optional fields to `ManualConnectionData` for SDK version tracking and permission management. The `UserInfoCreateRequest` now uses `UserAddress` type for the address field instead of the generic `Address` type.
* The SDK now supports enhanced insulin injection tracking with new optional fields for delivery mode, delivery form, and bolus purpose. Appointment rescheduling requests can now include appointment notes. Lab orders support clinical notes and new PSC lab provider options.
* Breaking change: The `ParsingJob.getJobId()` method has been removed. This field is no longer available in the ParsingJob response model. Remove any references to this method in your code.
* New optional fields added: `statusDetail` on ClientFacingOrderEvent, `orderTransaction` on LabResultsRaw, and `accessNotes` on PatientAddress models. The `GetOrderTransactionResponse.status` field is now a strongly-typed `OrderTransactionStatus` enum instead of a String.
* New query expression types now available: DerivedReadinessColumnExpr for derived readiness data and InsulinInjectionTimeseriesExpr for insulin injection timeseries data. The UserInfo class now uses a more specific UserAddress type, and the UsAddress class supports an optional access_notes field for delivery instructions.
* New optional `lab_slug` parameter available on `getMarkers()` method for improved lab filtering. Method documentation has been updated to clarify that `getMarkers()` excludes reflex markers while `getMarkersForLabTest()` includes all markers including reflex ones.
* The SDK now includes comprehensive logging support with configurable log levels and custom logger implementations. New CompendiumClient and LabAccountClient provide enhanced lab management capabilities. Added support for additional lab providers including Sonora Quest, CRL, Samsung Health, Tandem Source, Apero, and PVerify. Server-Sent Events support is now available through the SseEvent class.
* The SDK now includes a new compendium client with search and convert methods, and enhanced builder patterns for request classes with additional property methods.
* New compendium search and lab account management capabilities are now available. The SDK includes new request classes for searching and converting lab test data, plus a LabAccountClient for managing team lab accounts. Builder classes for existing request objects now support additional properties for enhanced flexibility.
* The SDK now includes lab account management capabilities through the new `RawLabAccountClient` with support for retrieving team lab accounts. Lab test orders now support an optional `clinicalNotes` field for providing additional clinical context.
* New `additionalProperty()` and `additionalProperties()` builder methods available on all request objects. These methods allow setting custom key-value pairs that will be included in API requests, providing flexibility for extensions and custom metadata.
* New `additionalProperty()` and `additionalProperties()` methods added to request builders, enabling custom properties to be included in API requests for enhanced flexibility.
* New builder methods available for setting additional properties on vitals request objects. The `additionalProperty()` and `additionalProperties()` methods allow customization of request parameters through a fluent builder interface.
* New `additionalProperty()` and `additionalProperties()` methods are now available on all vitals request builders, allowing developers to set custom key-value pairs for enhanced request flexibility.
* New `additionalProperty()` and `additionalProperties()` methods are now available on builder interfaces across the SDK for extended customization. The `AppointmentBookingRequest` also includes a new optional `appointmentNotes` field.
* The SDK now includes a new CanonicalCandidate type for representing canonical lab test candidates with LOINC and CPT code mappings. Enhanced builder classes now support dynamic property assignment through new additionalProperty() and additionalProperties() methods. Added appointment_notes field to appointment objects.
* The SDK now provides builder methods for setting additional properties on client-facing types, enabling more flexible JSON serialization and custom extensions.
* The SDK now supports builder methods for additional properties on all client-facing types. New `additionalProperty(String key, Object value)` and `additionalProperties(Map<String, Object> additionalProperties)` methods are available on builders. Added support for INSULIN_PUMP device source type.
* New additionalProperty() and additionalProperties() methods available on builder classes for client-facing types, enabling custom property support. The ClientFacingElectrocardiogramSourceType enum now includes INSULIN_PUMP as a new source type option.
* The SDK now provides builder methods for setting additional properties on client-facing model classes. New `additionalProperty()` and `additionalProperties()` methods are available on builders for enhanced extensibility when working with custom or unknown fields.
* New lab account and report parsing event types are now available. The SDK includes ClientFacingLabAccount for lab account metadata and webhook event classes for lab report parsing job lifecycle notifications (created/updated events).

Use `additionalProperty(key, value)` to set individual custom properties or `additionalProperties(map)` to set multiple properties at once on any model builder. A new `INSULIN_PUMP` source type is also available for menstrual cycle data.
