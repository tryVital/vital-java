## 2.0.0 - 2026-03-31
* The SDK now supports configurable HTTP logging through the new `logging()` method on `ClientOptions.Builder`. The deprecated `InputStream`-based file upload methods have been removed from lab report parser clients - use `File`-based methods instead. The `INSULIN_INJECTION` enum value has been removed from `IntervalTimeseriesExprTimeseries`.
* The SDK removes several file upload methods that accepted InputStream parameters from RawLabReportClient. New lab configuration options are available: CreateLabTestRequest now supports labAccountId and labSlug fields, LabTestsGetMarkersRequest adds labSlug filtering, and RequestAppointmentRequest includes appointmentNotes. The appointment availability API now uses a structured enum for lab selection instead of string values.
* The `teamId` parameter has been removed from all bulk operation classes (`BulkImportConnectionsBody`, `BulkPauseConnectionsBody`, `BulkTriggerHistoricalPullBody`, and `LinkListBulkOpsRequest`). Remove any calls to `getTeamId()` or `teamId()` methods on these classes. New optional fields have been added to `ManualConnectionData` for SDK version tracking and granted permissions, and the `Address` class now supports access notes.
* The SDK now supports enhanced insulin injection data with optional delivery metadata (delivery mode, form, and purpose), PSC labs appointment scheduling with Quest and Sonora Quest providers, and optional appointment notes for reschedule requests. New expression types have been added for data aggregation including derived readiness columns and insulin injection timeseries.
* The SDK now supports clinical notes on orders through the new `clinicalNotes` field. The `parentId` field has been removed from orders as it was deprecated. Order status information is now available through the enhanced `lastEvent` field which provides more detailed status tracking than the legacy `status` field.
* The SDK now includes an optional `accessNotes` field on patient address types for specifying delivery access instructions. New query expression types `DerivedReadinessColumnExpr` and `InsulinInjectionTimeseriesExpr` are now supported for enhanced data querying. Additional enum types `ResultMetadataGender` and `SearchMode` have been added for improved metadata handling.
* The UsAddress type now includes an optional accessNotes field for delivery instructions. The UserInfo type's address field now uses the more specific UserAddress type instead of the generic Address type.
* The SDK now supports optional iOS and Android SDK version tracking through custom headers in manual provider connections. Additionally, the deprecated team_id parameter has been removed from bulk operations.
* The SDK now includes two new clients: CompendiumClient and LabAccountClient for enhanced lab management operations. Additionally, comprehensive logging support has been added with configurable log levels (debug, info, warn, error) and custom logger implementations. New provider enums have been added including SONORA_QUEST, CRL, SAMSUNG_HEALTH, TANDEM_SOURCE, APERO, and PVERIFY. Order management now supports additional status tracking including corrected states and lab processing blocks.
* The SDK now supports Server-Sent Events (SSE) parsing with union type discrimination through the new SseEventParser utility class. A new CompendiumClient is available with search() and convert() methods for compendium operations. Request builders now include additionalProperty() and additionalProperties() methods for enhanced flexibility.
* New lab account management capabilities are now available through the `LabAccountClient`, allowing retrieval of team lab accounts. The SDK also introduces compendium search functionality with new request classes for lab test conversion and search operations.
* The SDK now includes lab account management capabilities through the new `RawLabAccountClient` class, which provides methods for retrieving team lab accounts with filtering options.
