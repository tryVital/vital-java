## 2.0.0 - 2026-03-30
* The SDK now supports configurable logging through the new `LogConfig` class and `ClientOptions.logging()` method. Logging is silent by default and requires explicit configuration. Additionally, advanced SSE streaming capabilities have been added with event-level discrimination support for discriminated unions where the discriminator is an SSE envelope field. Some legacy lab report parsing methods and enum values have been removed.
* Breaking change: The `RawLabReportClient.parserCreateJob()` method no longer accepts `InputStream` parameters. Use the `File`-based overloads instead. Additionally, the `lab` parameter in `LabTestsGetPscAppointmentAvailabilityRequest` now requires an `AppointmentPscLabs` enum value instead of a plain string, and the `teamId` field has been removed from `BulkExportConnectionsBody`.
* The SDK has removed the deprecated `teamId` field from bulk operation request classes. New optional fields have been added to support SDK version tracking (`vitalIosSdkVersion`, `vitalAndroidSdkVersion`) and permissions management (`grantedPermissions`) in manual connections. The `Address` type now includes an optional `accessNotes` field for additional location details.
* The SDK now supports additional metadata for insulin injection samples including delivery mode, delivery form, and bolus purpose. Appointment reschedule requests now accept optional appointment notes. Orders include a new clinical notes field. The deprecated parent_id field has been removed from orders.
* Breaking change: The `jobId` field has been removed from `ParsingJob`. Existing code accessing this field will need to be updated to remove references to `getJobId()` calls. The SDK now includes new optional fields for order status details, lab result order transactions, and patient address access notes. The `GetOrderTransactionResponse.status` field is now strongly typed as `OrderTransactionStatus` enum instead of a raw string.
* The SDK now supports derived readiness column expressions and insulin injection timeseries expressions in query operations. Additionally, the UsAddress type now includes an optional accessNotes field for delivery instructions.
* The lab tests API now supports filtering markers by lab slug using the new `lab_slug` parameter in `getMarkers()` methods. Method documentation has been improved to clarify that `getMarkers()` returns active/orderable markers excluding reflex markers, while `getMarkersForLabTest()` includes reflex markers.
* The SDK now supports mobile SDK version tracking through new optional headers (x-vital-ios-sdk-version and x-vital-android-sdk-version) in manual provider connections.
* The SDK now supports additional lab providers including Sonora Quest and CRL, along with new order statuses for corrected results and lab processing blocks. New logging capabilities allow configuring HTTP request/response logging with custom loggers and log levels. Two new client modules are available: CompendiumClient and LabAccountClient for accessing extended API functionality. Additional provider integrations include Samsung Health, Tandem Source, Apero, and PVerify.
* New CompendiumClient available with search() and convert() methods for accessing compendium data. The client supports both synchronous and asynchronous operations.
* New compendium API support for converting and searching lab test compendiums. The SDK now includes `ConvertCompendiumBody` and `SearchCompendiumBody` for lab test operations. Added `LabAccountClient` with `getTeamLabAccounts()` method for managing team lab accounts.
* The SDK now supports lab account management through the new `RawLabAccountClient` with `getTeamLabAccounts()` method. Additionally, clinical notes can now be included when creating lab test orders via the new `clinicalNotes` parameter in `CreateOrderRequestCompatible`.
* The SDK now supports additional properties on request objects through new builder methods. Developers can use `additionalProperty(key, value)` and `additionalProperties(map)` to include custom fields that will be passed through to the API.
* New builder methods `additionalProperty()` and `additionalProperties()` are now available on all request builders, enabling custom key-value pairs to be passed with API requests.
* The SDK now supports setting additional properties on vitals request objects through new `additionalProperty()` and `additionalProperties()` builder methods. These methods allow passing custom metadata or configuration options with vitals data requests.
* New `additionalProperty()` and `additionalProperties()` methods available on all vitals request builders to support custom field extensions.
* The SDK now provides better support for handling additional properties through new builder methods. All request and type builders now include `additionalProperty(String, Object)` and `additionalProperties(Map<String, Object>)` methods for setting custom properties. Additionally, the `AppointmentBookingRequest` class now supports an optional `appointmentNotes` field.
* New CanonicalCandidate type available for medical coding operations with support for LOINC codes, CPT codes, and confidence scoring. Builder classes now include additionalProperty() methods for enhanced API response handling.
* The SDK now provides enhanced builder methods for managing additional properties. All builder classes now include `additionalProperty(key, value)` and `additionalProperties(Map)` methods for flexible property management with fluent method chaining.
* New builder methods `additionalProperty()` and `additionalProperties()` are now available on client-facing types to customize additional properties. The SDK also adds support for insulin pump devices through the new `INSULIN_PUMP` device source type.
* New builder methods `additionalProperty()` and `additionalProperties()` are now available on response object builders for easier handling of custom properties. A new "insulin_pump" source type has been added for electrocardiogram data.
* The SDK now provides additionalProperty() and additionalProperties() builder methods on all model classes, allowing for custom field injection to support future API extensions and enhanced flexibility.
* The SDK now supports lab account management with the new `ClientFacingLabAccount` type, including configuration for billing, business units, and team allowlists. New webhook event types `ClientFacingLabReportParsingJobCreatedEvent` and `ClientFacingLabReportParsingJobUpdatedEvent` are available for tracking lab report parsing job lifecycle events.
* New `additionalProperty()` and `additionalProperties()` builder methods available on model classes for setting custom fields. The `INSULIN_PUMP` source type is now supported for menstrual cycle data.
* The SDK now provides builder methods for adding additional properties to model objects. Use `additionalProperty(key, value)` and `additionalProperties(map)` on any model builder to extend objects with custom data. The `ClientFacingSleepCycleSourceType` enum also adds support for `INSULIN_PUMP` devices.
* The SDK now provides convenient builder methods for setting additional properties on data types. Use `additionalProperty(key, value)` or `additionalProperties(map)` methods when building objects to include custom properties.
* New builder methods `additionalProperty()` and `additionalProperties()` are now available on all model builders, allowing you to set arbitrary key-value pairs on objects. A new `CompendiumSearchLabs` enum has been added with support for labcorp, quest, bioreference, and sonora_quest values.
* New `additionalProperty()` and `additionalProperties()` builder methods are now available on model builders for dynamic property extension. The SDK also adds support for lab test conversion operations with the new `ConvertCompendiumResponse` type and readiness data queries with `DerivedReadinessColumnExpr` types.
* New GetTeamLabAccountsResponse class available for retrieving team lab account data. Builder classes across all model types now support additionalProperty() and additionalProperties() methods for adding custom fields.
* New `additionalProperty()` and `additionalProperties()` methods are now available on all grouped data type builders, providing a consistent way to set custom properties during object construction.
* New builder methods `additionalProperty()` and `additionalProperties()` are now available on all grouped data type builders, allowing you to set custom properties when constructing API response objects.
* The SDK now supports additional property handling on all builder patterns through new `additionalProperty()` and `additionalProperties()` methods. New insulin injection timeseries capabilities are available via the `InsulinInjectionTimeseriesExpr` class, and lab account delegation workflows are supported with the new `LabAccountDelegatedFlow` enum.
* Builder classes now support setting additional properties via new `additionalProperty()` and `additionalProperties()` methods. The SDK also includes a new `OrderStatusDetail` enum for tracking order status error conditions.
* New PerLabCandidate type available for lab candidate matching functionality, including support for marker IDs, lab information, LOINC codes, CPT codes, and confidence scoring. Enhanced builder patterns across multiple types now support additional properties for better extensibility.
* New provider support added for Tandem Source and Samsung Health. The SDK now includes provider ID conversion capabilities through new response types that help map provider-specific identifiers to canonical formats with confidence scoring.
