## 2.0.0 - 2026-03-31
* The SDK now supports configurable logging through the ClientOptions builder. Use `.logging(LogConfig)` to enable HTTP request and response logging for better debugging and observability. The lab report parser methods that accepted InputStream parameters have been removed - use the File-based upload methods instead.
* Breaking: The `RawLabReportClient.parserCreateJob()` methods that accept `InputStream` parameters have been removed. Use the File-based overloads instead: replace `parserCreateJob(inputStream, filename)` with `parserCreateJob(file, request)`. New lab configuration options are now available including `labAccountId` and `labSlug` fields for lab test creation, and appointment scheduling now supports optional notes.
* The `teamId` field and `getTeamId()` method have been removed from `BulkImportConnectionsBody`, `BulkPauseConnectionsBody`, `BulkTriggerHistoricalPullBody`, and `LinkListBulkOpsRequest` classes. Code that accesses these members will need to be updated. New optional fields have been added to `ManualConnectionData` (SDK version tracking and granted permissions) and `Address` (access notes). The `UserInfoCreateRequest` now uses `UserAddress` instead of `Address` for the address field.
* The SDK now supports new aggregate expression types for derived readiness columns, awakenings values, and insulin injection timeseries. Insulin injection samples now include detailed delivery metadata (mode, form, bolus purpose). Appointment reschedule requests support optional notes, and PSC Labs appointment booking supports Quest and Sonora Quest options.
* The SDK now provides enhanced order management with new clinical notes support and detailed event information. A new `clinicalNotes` field is available on orders for storing clinical information, and the `lastEvent` field provides detailed information about the most recent order event. The `parentId` field has been removed from orders - existing code referencing this field will need to be updated.
* The SDK now supports an optional `access_notes` field on patient address types for delivery instructions. New query expression types are available for derived readiness data, sleep awakenings, and insulin injection timeseries.
* The SDK now supports an optional `accessNotes` field on `UsAddress` for providing delivery instructions or access notes. The `UserInfo` class now uses the more specific `UserAddress` type instead of the generic `Address` type for improved type safety.
* The lab tests markers API now supports an optional `lab_slug` parameter for improved lab-specific filtering. Method documentation has been enhanced to clarify that `getMarkers()` returns active and orderable markers only, while `getMarkersForLabTest()` includes reflex markers.
* The `getMarkers()` method now supports an optional `lab_slug` parameter for filtering markers by specific labs. Documentation has been updated to clarify that `getMarkers()` returns active and orderable markers only, while `getMarkersForLabTest()` includes reflex markers.
* The `teamId` parameter has been removed from bulk operations methods (`bulkImport`, `bulkExport`, `bulkPause`, and `bulkTriggerHistoricalPull`) in the Link client. Code using these methods with the `teamId` parameter will need to be updated to remove this parameter from the request objects.
* The SDK now supports additional laboratory providers (Sonora Quest, CRL), new health data providers (Samsung Health, Tandem Source), and comprehensive logging capabilities. New client methods `compendium()` and `labAccount()` are available on both sync and async clients. Configure logging using `VitalBuilder.logging()` or `AsyncVitalBuilder.logging()` with custom log levels and output handlers.
* The SDK now includes Server-Sent Events parsing capabilities through the new `SseEventParser` utility class. A new `CompendiumClient` is available with `search()` and `convert()` methods for compendium operations.
* New compendium search capabilities allow searching lab tests by CPT codes, LOINC sets, query strings, and target labs. Lab account management functionality provides access to team lab account information with filtering options.
* The SDK now supports lab account management with a new `RawLabAccountClient` that provides `getTeamLabAccounts()` for retrieving team lab accounts. Lab test orders can now include clinical notes through the optional `clinicalNotes` field in `CreateOrderRequestCompatible`.
* New `additionalProperty()` and `additionalProperties()` methods available on request builders for dynamic extensibility and custom data handling.
* The SDK now supports setting custom properties on request objects through new `additionalProperty()` and `additionalProperties()` methods available on all request builders.
* New builder methods available for setting additional properties on vitals request objects. The `additionalProperty()` and `additionalProperties()` methods allow passing custom metadata with vitals API requests.
* The SDK now supports setting additional properties on all vitals request objects through new builder methods. Use `.additionalProperty(key, value)` or `.additionalProperties(map)` when building vitals requests to include custom key-value pairs.
* New builder methods available for setting additional properties on request objects and model classes. The SDK now supports arbitrary key-value pairs through `additionalProperty()` and `additionalProperties()` methods on builders. A new `AwakeningsValueMacroExpr` class has been added for sleep data queries, and `AppointmentBookingRequest` now supports optional appointment notes.
* New CanonicalCandidate type available for lab result canonicalization with LOINC code mapping and confidence scoring. Enhanced builder patterns now support additional properties configuration across all response types.
* The SDK now supports setting additional properties on all client-facing types through new builder methods. Use `additionalProperty(key, value)` to set individual properties or `additionalProperties(map)` to set multiple properties at once.
* New builder methods for additional properties are now available across all model classes. Use `additionalProperty(key, value)` or `additionalProperties(map)` to include custom fields when building objects. The SDK also now supports `INSULIN_PUMP` as a device source type.
* The SDK now provides additional property builder methods across all client-facing model classes, allowing for easier handling of custom fields in API responses. A new `INSULIN_PUMP` source type is now available for electrocardiogram data.
* The SDK now supports adding custom properties to client-facing objects through new builder methods. Use `additionalProperty(key, value)` or `additionalProperties(map)` on builders to include additional fields during object construction.
* The SDK now supports lab report parsing job events through new `ClientFacingLabReportParsingJobCreatedEvent` and `ClientFacingLabReportParsingJobUpdatedEvent` classes. These enable webhook subscriptions for real-time notifications when lab report parsing jobs are created or updated. A new `ClientFacingLabAccount` type is also available for managing lab account configurations.
* The SDK now provides enhanced builder methods for handling additional properties across all client-facing types. New `additionalProperty()` and `additionalProperties()` methods allow flexible management of custom fields in API responses. The `ClientFacingMenstrualCycleSourceType` enum now includes support for insulin pump data sources.
* The SDK now includes a recovery readiness score field in sleep data, providing access to recovery/readiness metrics from Oura, Whoop, and Ultrahuman devices. Builder classes also now support convenient methods for adding additional properties.
* New builder methods for handling additional properties are now available on all model classes. Use `additionalProperty(String key, Object value)` to set individual properties or `additionalProperties(Map<String, Object>)` to set multiple properties at once.
* The SDK now supports setting additional properties on all client-facing model objects through new builder methods `additionalProperty()` and `additionalProperties()`, enabling better forward compatibility and extensibility.
* The SDK now supports compendium lab test conversion functionality with new `CompendiumSearchLabs` enum and `ConvertCompendiumResponse` class. Enhanced builder patterns across model classes now include `additionalProperty()` and `additionalProperties()` methods for flexible property handling.
* New `additionalProperty()` and `additionalProperties()` methods added to all builder classes, enabling flexible handling of additional JSON properties in API responses.
* The SDK now supports setting custom additional properties on grouped data type builders. New `additionalProperty()` and `additionalProperties()` methods are available on builder classes for all grouped data types including calories, heart rate, distance, and other health metrics.
* The SDK now supports setting additional properties on grouped response builders. New optional `additionalProperty()` and `additionalProperties()` methods are available on all grouped response builders.
* The SDK now provides builder methods for setting additional properties on data classes via `additionalProperty()` and `additionalProperties()` methods. New support for insulin injection timeseries queries is available through the `InsulinInjectionTimeseriesExpr` class.
* Enhanced builder patterns with additional properties support. Builder classes now include `additionalProperty()` and `additionalProperties()` methods for setting custom key-value pairs on model objects. Also added new `OrderStatusDetail` enum for order status error handling.
* The SDK now includes a new `PerLabCandidate` model for laboratory test candidate matching, providing detailed information about test markers, LOINC codes, CPT codes, and confidence scoring. Additionally, builder patterns have been improved across multiple models with enhanced support for additional properties.
