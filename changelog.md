## 2.0.0 - 2026-03-30
* The SDK now supports event-level discrimination for Server-Sent Events (SSE) streams through new `fromSseWithEventDiscrimination` methods. Configurable logging has been added via the `LogConfig` class in `ClientOptions`. The deprecated `INSULIN_INJECTION` enum value and InputStream-based file upload methods have been removed from the lab report parser.
* Removed deprecated InputStream-based file upload methods from lab report parsing. Use File-based alternatives instead. Added new optional lab configuration fields (labAccountId, labSlug) for better lab test customization and appointment management.
* The deprecated `team_id` parameter has been removed from bulk connection operations including `BulkImportConnectionsBody`, `BulkPauseConnectionsBody`, `BulkTriggerHistoricalPullBody`, and `LinkListBulkOpsRequest`. Update your code to remove any `team_id` parameter usage from these operations. The SDK now also supports optional SDK version tracking and granted permissions fields in manual connections.
* The SDK now supports additional insulin injection tracking fields including delivery mode, delivery form, and bolus purpose. Appointment reschedule requests can now include optional notes, and order management includes a new clinical notes field.
* The SDK now supports enhanced order tracking with optional status details in order events, order transaction information in lab results, and access notes for patient addresses. Order transaction status is now returned as a proper enum type instead of a string.
* The SDK now supports querying derived readiness data and insulin injection timeseries through new expression types in the query API. Additional enums for gender metadata and search modes are now available, and the address type system has been enhanced with access notes support.
* The `getMarkers()` method now supports an optional `lab_slug` query parameter for filtering markers by lab slug. Method documentation has also been improved to clarify the distinction between `getMarkers()` (active/orderable markers only) and `getMarkersForLabTest()` (includes reflex markers).
* The SDK now supports tracking mobile SDK versions through x-vital-ios-sdk-version and x-vital-android-sdk-version headers when connecting manual providers. The team_id parameter has been removed from bulk operations (bulk_import, bulk_trigger_historical_pull, bulk_export, bulk_pause) as it is no longer supported.
* The SDK now supports additional healthcare lab providers including Sonora Quest and CRL. New client modules `CompendiumClient` and `LabAccountClient` are available for expanded functionality. Added configurable logging with `LogConfig` - call `.logging()` on the builder to enable debug output or use a custom logger implementation.
* The SDK now includes a new Compendium API client with methods for searching and converting compendium data. A new SseEventParser utility class has been added for handling Server-Sent Events with discriminated union support. Request builders now support additional property methods for enhanced customization.
* The SDK now includes lab account management capabilities through the new `LabAccountClient` with support for retrieving team lab accounts. Additionally, new request classes `ConvertCompendiumBody` and `SearchCompendiumBody` have been added for compendium search operations.
* The SDK now supports lab account management through the new RawLabAccountClient, enabling retrieval of team lab accounts with filtering capabilities. Additionally, clinical notes can now be included when creating lab test orders via the optional clinical_notes field.
* New `additionalProperty()` and `additionalProperties()` builder methods are now available on all request classes. These methods allow you to add custom key-value pairs to requests for future extensibility.
* The SDK now supports adding custom properties to request objects via new `additionalProperty()` and `additionalProperties()` methods on request builders. This allows for greater flexibility when working with API extensions or custom fields.
* New builder methods `additionalProperty()` and `additionalProperties()` available on all vitals request classes, enabling custom key-value pairs to be included in requests.
* The SDK now supports adding additional properties to vitals request objects via new builder methods `additionalProperty()` and `additionalProperties()`.
* The SDK now supports setting additional properties on request and type objects through new `additionalProperty()` and `additionalProperties()` builder methods. A new optional `appointmentNotes` field is also available on appointment booking requests.
* New CanonicalCandidate type available for medical test LOINC code mapping with popularity scoring and confidence metrics. Enhanced builder patterns now include additionalProperty() methods for better extensibility.
* The SDK now provides enhanced builder methods for setting additional properties on event and model objects. Two new methods are available on all builders: `additionalProperty(String key, Object value)` for setting individual properties and `additionalProperties(Map<String, Object> additionalProperties)` for bulk property assignment.
* The SDK now supports setting additional properties on all client-facing types through new builder methods `additionalProperty()` and `additionalProperties()`. A new device source type `INSULIN_PUMP` is also available in the `ClientFacingDeviceSourceType` enum.
* The SDK now supports additional property handling through new `additionalProperty()` and `additionalProperties()` methods on builder classes. A new `INSULIN_PUMP` source type is available for electrocardiogram data.
* New builder methods `additionalProperty()` and `additionalProperties()` are available on all client-facing type builders, allowing custom fields to be included during object construction. Two new enum types `ClientFacingInsulinInjectionSampleDeliveryMode` and `ClientFacingInsulinInjectionSampleBolusPurpose` have been added for insulin injection samples.
* The SDK now supports lab report parsing job webhook events with new `ClientFacingLabReportParsingJobCreatedEvent` and `ClientFacingLabReportParsingJobUpdatedEvent` classes. Additionally, comprehensive lab account management is available through the new `ClientFacingLabAccount` class, providing detailed control over laboratory partnerships and billing configurations.
* The SDK now supports setting custom additional properties on API objects through new builder methods `additionalProperty()` and `additionalProperties()`. A new `INSULIN_PUMP` source type has been added to menstrual cycle data sources.
* The SDK now includes additionalProperty methods on builder classes, allowing developers to set custom properties when constructing model objects. Use `builder.additionalProperty(key, value)` or `builder.additionalProperties(map)` to add extra data to any model.
* Enhanced builder patterns for all ClientFacing model classes now include `additionalProperty()` and `additionalProperties()` methods, allowing developers to add custom key-value pairs to model instances for extended functionality.
* New builder methods `additionalProperty()` and `additionalProperties()` are now available on ClientFacing types and response classes, allowing you to set custom properties during object construction. A new `CompendiumSearchLabs` enum has also been added with lab provider constants.
* New types available for lab test conversion (ConvertCompendiumResponse) and readiness data analysis (DerivedReadinessColumnExpr). Additional property builder methods added to model classes for enhanced extensibility.
* The SDK now supports custom additional properties on builder objects through new `additionalProperty()` and `additionalProperties()` methods. A new `GetTeamLabAccountsResponse` class has been added for team lab account management operations.
* New `additionalProperty()` and `additionalProperties()` methods are now available on builder classes for grouped data types, allowing dynamic property assignment during object construction.
* The SDK now supports setting additional properties on Grouped response objects through enhanced builder methods. New `additionalProperty()` and `additionalProperties()` methods are available on all Grouped* response type builders, enabling programmatic customization of response objects during construction.
* New builder methods for additional properties are now available on model classes, enabling more flexible object construction. The SDK also adds support for insulin injection timeseries queries and new lab account delegation workflow options.
* Enhanced builder pattern support with new additionalProperty() and additionalProperties() methods on model classes. This allows setting custom properties during object construction using a fluent interface. Also added OrderStatusDetail enum for order status handling.
* New `PerLabCandidate` class available for lab marker matching operations, providing confidence scoring and comprehensive LOINC code support.
* The SDK now supports two new health data providers: Tandem Source and Samsung Health. New provider ID conversion capabilities are available through the ProviderIdConversionResponse type. Lab result metadata now includes gender information via the ResultMetadata type.
