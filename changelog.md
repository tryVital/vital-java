## 2.0.0 - 2026-03-20
* The SDK now supports optional logging configuration through ClientOptions.Builder.logging(). Logging is disabled by default. The Stream class adds new SSE event discrimination capabilities via fromSseWithEventDiscrimination() methods for handling discriminated union streaming responses.
* The SDK removes four InputStream-based `parserCreateJob` method overloads from `RawLabReportClient`. These methods have been replaced with File-based alternatives. New lab configuration options are available including `labAccountId` and `labSlug` fields for lab test requests, and `appointmentNotes` for appointment scheduling.
* New optional fields added to ManualConnectionData for mobile SDK version tracking (vitalIosSdkVersion, vitalAndroidSdkVersion) and permission management (grantedPermissions). Address type now includes accessNotes for delivery instructions. AppointmentRescheduleRequest supports appointmentNotes, and ClientFacingOrder includes clinicalNotes. Enhanced aggregate expressions now support DerivedReadinessColumnExpr for readiness data analysis.
* The `jobId` field has been removed from `ParsingJob`. Existing code accessing this field will need to be updated. Additionally, `GetOrderTransactionResponse.status` now returns an `OrderTransactionStatus` enum instead of a raw string. New optional fields are available: `statusDetail` on `ClientFacingOrderEvent`, `orderTransaction` on `LabResultsRaw`, and `accessNotes` on all `PatientAddress` types.
* The SDK now supports DerivedReadinessColumnExpr for query operations and adds an optional accessNotes field to UsAddress for delivery instructions. UserInfo.address type has changed from Address to UserAddress.
* The SDK now supports configurable logging with custom log levels and backends. New provider constants added for Sonora Quest, Samsung Health, and Tandem Source integrations. Additional lab status values and corrected order states are now available. New CompendiumClient and LabAccountClient provide access to additional API endpoints.
* The SDK now includes a new CompendiumClient for searching and converting compendium resources, with full support for synchronous and asynchronous operations. Additionally, a new SseEventParser utility class is available for parsing Server-Sent Events with discriminated union support.
* New compendium search and conversion capabilities available for lab test management. The SDK now includes methods to search lab tests by CPT codes, LOINC hashes, or text queries, convert tests between different lab providers, and manage team lab accounts through the new LabAccountClient.
* The SDK now supports lab account management through the new `RawLabAccountClient` with methods to retrieve team lab accounts. Lab test orders now include an optional `clinical_notes` field for additional context.
* New `additionalProperty()` and `additionalProperties()` methods are now available on all request builders, allowing you to add custom properties that are not explicitly defined in the API schema.
* New `additionalProperty()` and `additionalProperties()` methods are now available on all request builder classes, allowing users to set custom properties on request objects.
* New builder methods for dynamic properties on vitals request classes. The SDK now supports `additionalProperty()` and `additionalProperties()` methods on all vitals request builders, allowing custom key-value pairs to be added to requests.
* New `additionalProperty()` and `additionalProperties()` methods available on all vitals request builders, allowing custom key-value pairs to be included in API requests.
* New `additionalProperty()` and `additionalProperties()` builder methods are now available on request objects and response types, allowing custom key-value pairs to be included. The `AppointmentBookingRequest` now supports an optional `appointmentNotes` field for adding notes to appointment bookings.

All model builders now support setting additional properties via `additionalProperty()` and `additionalProperties()` methods for improved extensibility.
