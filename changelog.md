## 2.0.0 - 2026-04-08
* Several breaking changes are included in this release:
* The `parserCreateJob(InputStream, String)`, `parserCreateJob(InputStream, String, MediaType)`, `parserCreateJob(InputStream, String, RequestOptions)`, and `parserCreateJob(InputStream, String, MediaType, RequestOptions)` overloads have been removed from both `LabReportClient` and `AsyncLabReportClient`. Callers must migrate to the `File`-based overloads (e.g., `parserCreateJob(File, BodyCreateLabReportParserJob)`).
* The `INSULIN_INJECTION` constant has been removed from the `IntervalTimeseriesExprTimeseries` enum. Any exhaustive switch statements referencing this value must be updated.
* New in this release: the SDK now supports configurable request/response logging via a `LogConfig` instance passed to `ClientOptions.builder().logging(logConfig)`. The SDK is silent by default — no log output unless explicitly configured. Additionally, `Stream` now supports SSE event-level discrimination for deserializing discriminated-union payloads from SSE streams.
* Several breaking changes have been made to request types in this release.
* **`LabTestsGetPscAppointmentAvailabilityRequest`**: The `lab` field type has changed from `String` to `AppointmentPscLabs`. Callers must update to pass an `AppointmentPscLabs` enum value instead of a raw string.
* **`BulkExportConnectionsBody`**: The `teamId` field (`Optional<LinkBulkExportRequestTeamId>`) and its builder methods (`teamId(...)`) have been removed. Remove any calls to `.teamId(...)` in your builder chains.
* **`RawLabReportClient`**: The four `parserCreateJob` overloads that accepted `InputStream` (with and without `MediaType`/`RequestOptions`) have been removed. Use the `File`-based overloads instead.
* New optional fields have also been added: `labAccountId` and `labSlug` on `CreateLabTestRequest`, `labSlug` on `LabTestsGetMarkersRequest`, and `appointmentNotes` on `RequestAppointmentRequest`.
* Several breaking changes have been made to request and type classes in this release.
* The `teamId` field (and associated getter/builder methods) has been removed from `BulkImportConnectionsBody`, `BulkPauseConnectionsBody`, `BulkTriggerHistoricalPullBody`, and `LinkListBulkOpsRequest`. Callers that set or read `teamId` on these types must remove those usages.
* The `address` field in `UserInfoCreateRequest` has changed from `Address` to `UserAddress`; update any code that passes an `Address` instance or calls `getAddress()` expecting `Address`. Additionally, `ManualConnectionData` now supports optional `vitalIosSdkVersion`, `vitalAndroidSdkVersion`, and `grantedPermissions` fields, and `Address` gains a new optional `accessNotes` field.
* The `AggregateExprArg.Visitor<T>` interface now requires three additional `visit()` method implementations: `visit(DerivedReadinessColumnExpr)`, `visit(AwakeningsValueMacroExpr)`, and `visit(InsulinInjectionTimeseriesExpr)`. Consumers must add these methods to any class implementing the interface to avoid compile errors.
* Additionally, `ClientFacingInsulinInjectionSample` now exposes optional `deliveryMode`, `deliveryForm`, and `bolusPurpose` fields, `AppointmentRescheduleRequest` gains an optional `appointmentNotes` field, and two new enums (`AppointmentPscLabs`, `ClientFacingInsulinInjectionSampleDeliveryForm`) have been added.
* Several breaking changes have been made to order and lab result types:
* **`ClientFacingOrder`**: The `parentId` field and its builder methods (`parentId(...)`) have been removed. A new required `lastEvent` field (`ClientFacingOrderEvent`) has been added — existing builder chains must now call `.lastEvent(...)` after `.updatedAt(...)`. A new optional `clinicalNotes` field is also available.
* **`ClientFacingOrderEvent`**: A new optional `statusDetail` (`OrderStatusDetail`) field has been added to the event type.
* **`GetOrderTransactionResponse`**: The `status` field has changed from `String` to the strongly-typed `OrderTransactionStatus` enum. Update any code that assigns or compares this field as a `String`.
* **`HistoricalPullCompleted`**: The `isFinal` builder step has been removed; `getIsFinal()` now always returns `true`. Remove calls to `.isFinal(...)` in builder chains.
* **`LabResultsRaw`**: A new optional `orderTransaction` (`ClientFacingOrderTransaction`) field has been added.
* **`LinkBulkExportRequestTeamId`** has been removed and replaced by the new `LabAccountStatus` enum (`ACTIVE`, `PENDING`, `SUSPENDED`).
* **Breaking changes in this release:**
* **`ParsingJob`**: The `jobId` field and its required builder stage (`JobIdStage`) have been removed. Remove any `.jobId(...)` calls from `ParsingJob.Builder` chains.
* **`Placeholder`**: The required `placeholder(boolean)` builder step has been removed. `getPlaceholder()` now returns `Boolean` (always `true`) instead of `boolean`. Call `.builder().build()` directly.
* **`QueryGroupByItem.Visitor<T>`**: Three new abstract `visit(...)` methods have been added to the `Visitor` interface — `visit(DerivedReadinessColumnExpr)`, `visit(AwakeningsValueMacroExpr)`, and `visit(InsulinInjectionTimeseriesExpr)`. All existing `Visitor` implementations must implement these new methods.
**New capabilities:**
* `PatientAddress`, `PatientAddressCompatible`, and `PatientAddressWithValidation` now expose an optional `getAccessNotes()` field (mapped to `access_notes`) and a corresponding builder setter.
* `QueryGroupByItem` now supports three new union members: `DerivedReadinessColumnExpr`, `AwakeningsValueMacroExpr`, and `InsulinInjectionTimeseriesExpr`.
* Several breaking changes have been introduced in this release:
* **`QuerySelectItem.Visitor`** now requires three additional `visit()` methods — `visit(DerivedReadinessColumnExpr)`, `visit(AwakeningsValueMacroExpr)`, and `visit(InsulinInjectionTimeseriesExpr)` — so any existing `Visitor` implementation will fail to compile until these methods are added.
* **`UserInfo.getAddress()`** now returns `UserAddress` instead of `Address`; callers and builder code that pass or receive `Address` must be updated to use `UserAddress`.
* **`UserRefreshErrorResponse` and `UserRefreshSuccessResponse`** no longer expose a `success` builder stage: `builder()` now returns `UserIdStage` directly, and the `success(boolean)` setter has been removed. The `getSuccess()` return type changed from `boolean` to `Boolean`. Update any builder chains that previously called `.success(...)`.
* **`LinkBulkImportRequestTeamId`** (package `com.vital.api.resources.link.types`) has been replaced by `ResultMetadataGender` (package `com.vital.api.types`), and **`LinkListBulkOpsRequestTeamId`** has been replaced by `SearchMode`. Update all import statements and references accordingly.
* The `getMarkers` method now supports an optional `lab_slug` filter parameter via `LabTestsGetMarkersRequest`, allowing consumers to filter markers by lab slug. Javadoc for `getMarkers` and `getMarkersForLabTest` has been updated to clarify that `getMarkers` returns only active and orderable markers (excluding reflex markers), while `getMarkersForLabTest` also includes associated reflex markers.
* The `getMarkers` method on `LabTestsClient` now supports filtering by lab slug via the new `lab_slug` query parameter on `LabTestsGetMarkersRequest`. Additionally, the `getMarkersForLabTest` methods now include Javadoc clarifying that reflex markers are included in their results.
* The SDK now supports sending optional `x-vital-ios-sdk-version` and `x-vital-android-sdk-version` headers when connecting a manual provider via the link API. The `team_id` query parameter has been removed from bulk link operations (`bulk_import`, `bulk_trigger_historical_pull`, `bulk_export`, `bulk_pause`).
* The SDK now supports configurable logging via a new `LogConfig` API — logging is silent by default and can be enabled on `VitalBuilder` or `AsyncVitalBuilder` using `.logging(LogConfig.builder().silent(false).build())`. Two new resource clients are available: `CompendiumClient` (via `Vital.compendium()`) and `LabAccountClient` (via `Vital.labAccount()`). Several enums have been expanded with new values: `SONORA_QUEST` in `AppointmentProvider`, `ClientFacingLabs`, and `Labs`; `CRL` in `Labs`; `SAMSUNG_HEALTH` in `ManualProviders`; `CORRECTED` and `LAB_PROCESSING_BLOCKED` in `OrderLowLevelStatus`; five new corrected/blocked statuses in `OrderStatus`; `TANDEM_SOURCE` in `PasswordProviders`; and `APERO` and `PVERIFY` in `PayorCodeExternalProvider`.
* The SDK now includes a new `CompendiumClient` (and async variant) with `search()` and `convert()` methods for interacting with the Compendium API. A new `SseEventParser` utility has been added to support parsing Server-Sent Events with discriminated unions. Additionally, `additionalProperty` and `additionalProperties` builder methods are now available on request builder interfaces across multiple resources (Activity, Body, Aggregate).
* The SDK now includes a new `LabAccountClient` (and async equivalent) exposing `getTeamLabAccounts()` for retrieving team lab account information. Two new request body classes, `SearchCompendiumBody` and `ConvertCompendiumBody`, are available for compendium search and convert operations.
