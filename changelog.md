## 2.0.0 - 2026-03-18
* The SDK now supports configurable logging through the new `logging()` method on `ClientOptions.Builder`. Server-Sent Events (SSE) streams now support event-level discrimination via new `fromSseWithEventDiscrimination()` factory methods on the `Stream` class, enabling proper handling of discriminated unions where the discriminator is in the SSE envelope. Lab report parser methods that accepted `InputStream` parameters have been removed in favor of `File`-based methods.
* The SDK removes InputStream-based file upload methods for lab report parsing in favor of File-based uploads that support multiple files. New optional parameters are available for lab test requests: lab account ID and lab slug for test creation, lab slug for marker queries, and appointment notes for scheduling. The appointment availability request now uses a typed enum for lab selection instead of a string.
* The SDK now supports additional optional fields across multiple data types including mobile SDK version tracking in ManualConnectionData, clinical notes in orders, appointment notes for rescheduling, and access notes for addresses. New DerivedReadinessColumnExpr support has been added for enhanced query capabilities.
* Breaking changes to GetOrderTransactionResponse and ParsingJob models. The `status` field in GetOrderTransactionResponse now returns an OrderTransactionStatus enum instead of a string - update code that directly compares status strings to use the enum values. The `jobId` field has been removed from ParsingJob - remove any references to this field in your code. Additionally, several models now include new optional fields: LabResultsRaw includes orderTransaction context, and address models include accessNotes for delivery instructions.
* The SDK now supports derived readiness column expressions in query select items through the new `DerivedReadinessColumnExpr` type. Additionally, the `UsAddress` type now includes an optional `access_notes` field for capturing delivery instructions or access information.
* The `getMarkers()` method now accepts an optional `lab_slug` parameter for filtering markers by lab. Method documentation has been updated to clarify that reflex markers are excluded from standard listings, while `getMarkersForLabTest()` includes both regular and reflex markers.
* The SDK now supports filtering markers by lab_slug parameter in the getMarkers method. Additionally, mobile SDK version tracking headers are now included in manual provider connection requests.
* The SDK now includes platform-specific version headers (x-vital-ios-sdk-version and x-vital-android-sdk-version) when creating manual provider links, enabling better tracking of mobile SDK usage.
* The SDK now supports configurable logging with the new `logging()` method on client builders. Two new client modules are available: `CompendiumClient` and `LabAccountClient`. Additional enum values have been added for various providers and status types including Sonora Quest support and corrected order statuses.
* New CompendiumClient available for searching and converting compendium data. The SDK now includes comprehensive Server-Sent Events parsing utilities with support for discriminated unions.
* New compendium search and convert functionality is now available through ConvertCompendiumBody and SearchCompendiumBody request classes. The SDK also introduces lab account management capabilities with the new LabAccountClient and getTeamLabAccounts method.
* New lab account management capabilities available through `RawLabAccountClient.getTeamLabAccounts()` method. Lab test orders now support optional clinical notes via the `clinicalNotes` field in `CreateOrderRequestCompatible`.
* Request builders now support adding custom properties through new `additionalProperty(String key, Object value)` and `additionalProperties(Map<String, Object> additionalProperties)` methods. These methods enable flexible extension of requests while maintaining full backward compatibility.
* New `additionalProperty()` and `additionalProperties()` methods are now available on all request builders, allowing you to set custom properties at runtime for enhanced SDK extensibility.
* New builder methods `additionalProperty()` and `additionalProperties()` are now available on all vitals request classes. These methods allow you to add custom key-value pairs to requests for extended functionality and metadata support.
* The SDK now supports adding custom properties to vitals request objects through new `additionalProperty()` and `additionalProperties()` builder methods.
* The SDK now supports setting additional properties on request builders through new `additionalProperty()` and `additionalProperties()` methods. The `AppointmentBookingRequest` class includes a new optional `appointmentNotes` field for adding notes to appointment bookings.
* The SDK now includes enhanced builder methods for handling additional properties. All type builders now support `additionalProperty()` and `additionalProperties()` methods for setting dynamic fields, improving forward compatibility. A new `CanonicalCandidate` type has been added with comprehensive medical coding support.
* The SDK now supports appointment notes through a new `appointmentNotes` field on `ClientFacingAppointment` objects. The field is accessible via the `getAppointmentNotes()` method and can be set using the builder pattern.
* Builder classes now support additional properties through new `additionalProperty()` and `additionalProperties()` methods. This enables extending objects with custom fields while maintaining fluent builder patterns.
* New `additionalProperty()` and `additionalProperties()` methods are now available on all model builder classes. These methods allow you to set custom properties on model objects during construction using the fluent builder API.
* The SDK now supports setting additional properties on builder objects through new `additionalProperty()` and `additionalProperties()` methods. This enables handling of arbitrary key-value pairs that may be returned from the API but are not explicitly modeled in the type definitions.
* New lab report parsing job event types are now available for webhook subscriptions. The SDK now supports ClientFacingLabReportParsingJobCreatedEvent and ClientFacingLabReportParsingJobUpdatedEvent for tracking lab report processing status. Additionally, all builder classes now support setting additional properties for enhanced JSON serialization flexibility.

* New builder methods `additionalProperty()` and `additionalProperties()` are now available on all client-facing model builders, enabling programmatic customization with arbitrary key-value pairs.
* New builder methods `additionalProperty()` and `additionalProperties()` are now available on all client-facing type builders, enabling developers to set custom metadata on API objects.
* Builder methods now support additional properties via `additionalProperty()` and `additionalProperties()` methods on all data type builders. This enables more flexible construction of objects with dynamic or extension fields.
